# Coverage Audit

This file answers a simple question: does this repo cover what is needed for serious Java DSA interview preparation?

Short answer: **yes for the core coding interview path**. The repo now covers the major patterns, the important data structures, the common problem types, and the revision process needed to retain them.

---

## Coverage Verdict

This curriculum is strong enough for a focused one-month sprint because it includes:

- a Java fundamentals layer
- topic-by-topic concept notes
- Java pattern implementations
- a day-by-day study plan
- a problem tracker
- a spaced-repetition revision queue
- interview communication notes
- common mistake checklists

It is not only a list of problems. It is a learning system.

---

## Tier 1 -- Non-Negotiable Topics

These must become comfortable. Do not skip them.

- Big-O analysis
- Java arrays, strings, lists, maps, sets, queues, stacks, priority queues
- Arrays and HashMap
- Frequency counting
- Prefix sums
- Two pointers
- Sliding window
- Binary search
- Binary search on answer
- Stack and monotonic stack
- Linked list pointer operations
- Tree DFS and BFS
- Graph BFS and DFS
- Dynamic programming basics

Most coding interviews are built from these patterns.

---

## Tier 2 -- Very Important Topics

These appear frequently and should be practiced after the Tier 1 base is moving.

- Backtracking
- BST properties
- Heaps and top-k
- Intervals
- Greedy
- Union-Find
- Tries
- Matrix traversal and transformation
- Design-style coding problems
- Bit manipulation basics
- Common math patterns

These topics often separate a decent interview from a strong one.

---

## Tier 3 -- Stretch Topics

Know the idea, but do not let these steal time from the core sprint.

- Segment tree
- Fenwick tree
- Advanced string algorithms
- Minimum spanning tree algorithms
- Bellman-Ford and Floyd-Warshall
- Advanced DP
- Heavy combinatorics
- Geometry

These are useful, but they are not the best use of time in the first 30 days unless a target company is known to ask them.

## What To Skip in the First 30 Days

Skip these unless there is extra time:

- Segment tree and Fenwick tree
- Suffix arrays
- Heavy bitmask DP
- Rare math puzzles
- Deep geometry
- Full system design deep dives

For backend interviews, learn system design separately. This sprint is mainly for coding rounds and data-structure design questions.

---

## Success Criteria

By the end of the sprint, aim for:

- 80 to 95 attempted problems
- 55 to 65 clean independent solves
- 20+ successful revisits
- 3 timed sessions or mocks
- clear notes on weak patterns
- confidence writing Java templates without looking

The most important metric is not total count. It is whether you can solve, explain, test, and redo.
